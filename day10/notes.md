# Day 10 - Sorting Algorithms (Theory)

## What is Sorting?
Sorting is the process of arranging data in a specific order (ascending or descending).

---

## Why Sorting?
- Makes searching faster
- Organizes data
- Used in many algorithms

---

## Types of Sorting

- Internal Sorting → Data fits in memory
- External Sorting → Data is large (stored in disk)

---

## Common Sorting Algorithms

1. Bubble Sort
- Compare adjacent elements and swap
- Time Complexity → O(n²)

---

2. Selection Sort
- Select smallest element and place it correctly
- Time Complexity → O(n²)

---

3. Insertion Sort
- Insert element at correct position
- Time Complexity → O(n²)

---

4. Merge Sort
- Divide and merge
- Time Complexity → O(n log n)

---

5. Quick Sort
- Divide using pivot
- Time Complexity → O(n log n) (average)

---

## Comparison

| Algorithm       | Time Complexity |
|----------------|----------------|
| Bubble Sort    | O(n²)          |
| Selection Sort | O(n²)          |
| Insertion Sort | O(n²)          |
| Merge Sort     | O(n log n)     |
| Quick Sort     | O(n log n)     |

---

## Key Idea

- Simple sorts → easy but slow  
- Advanced sorts → faster but complex  

---

## Important Points

- Merge sort is stable  
- Quick sort is faster in practice  
- Sorting improves searching efficiency
