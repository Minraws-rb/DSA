# Day 6 - Recursion 

## What is Recursion?
Recursion is a technique where a function calls itself to solve a problem.


## Basic Idea
A recursive function breaks a problem into smaller subproblems of the same type.


## Components of Recursion

1. Base Case  
   → Condition where recursion stops  

2. Recursive Case  
   → Function calls itself with smaller input  


## Example Concept

Factorial:
n! = n × (n-1)!

Example:
5! = 5 × 4 × 3 × 2 × 1



## Recursion vs Iteration

| Recursion | Iteration |
|----------|----------|
| Uses function calls | Uses loops |
| Uses stack memory | No extra stack |
| Easier to write | More efficient |


## Recursion and Stack

- Each function call is stored in stack memory  
- When function finishes, it is removed from stack  



## Types of Recursion

1. Direct recursion → function calls itself  
2. Indirect recursion → function calls another function  


## Applications of Recursion

- Factorial calculation  
- Fibonacci series  
- Tower of Hanoi  
- Tree traversal
